# Contributing

Contributions should improve reliability, clarity, portability, or safety.

Before opening a pull request:

1. Explain the failure mode the change addresses.
2. Keep changes project-neutral unless they live in an example/template.
3. Avoid proprietary prompts, secrets, customer data, or private internal details.
4. Run `npm run validate`.
5. Include before/after examples when changing agent behavior.

Good contributions are small, evidence-backed, and easy to review.
