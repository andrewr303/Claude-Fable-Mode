# /fable-mode

Use Fable Mode for the current task.

Instructions:

1. Classify task risk and choose the lightest adequate intensity.
2. Inspect the real repo/sources before editing or claiming.
3. Write a concise stage map with artifacts and checks.
4. Use subagents only for independent work, each with a return schema and anti-fabrication rule.
5. Verify each stage with checks that can fail.
6. Inspect rendered/generated artifacts before calling them complete.
7. Ask approval before irreversible, outward-facing, production, auth, money, legal/compliance, or data-loss actions.
8. Final handoff must separate changed files, verification, not-run items, risks, and next action.
