# Fable Mode for Codex

Use the repository-level `AGENTS.md` for full behavior. This plugin provides a portable `fable-mode` skill for Codex-style runtimes.

Core behavior:

- Inspect the real system before editing.
- Create a concise stage map for complex work.
- Delegate only independent work, and only with a return schema.
- Verify with checks that can fail.
- Inspect rendered/generated artifacts before declaring completion.
- Ask approval before irreversible, outward-facing, production, auth, money, legal/compliance, or data-loss actions.
- Separate verified facts, assumptions, not-run checks, and remaining risks in the final handoff.
